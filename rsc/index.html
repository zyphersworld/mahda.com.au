    What this does
    

      
Server-rendered React (Next.js / RSC) pages serialise their data into a sequence of
      self.__next_f.push([1,"…"]) calls split across several <script> tags.
      The chunk index is not stable between pages, so this tool concatenates every chunk, rebuilds the
      payload, and resolves your dotted property path against it — regardless of which
      script tag the data landed in.

      
When a resolved value is a stream reference (e.g. "$f"), the resolver follows it to
      the row that holds the referenced value. Output is returned as raw data: a JSON
      value with its type, key count, and length. It is an inspection and debugging aid, not a viewer or
      renderer for any particular content type.

      
Relay: a file opened locally cannot fetch a different origin (the browser's
      same-origin policy). The relay forwards the request and adds the CORS header the browser needs.
      Public relays are best-effort and rate-limited; if one fails, try another, or host this file on the
      origin you're inspecting and choose None.

    

  


  

    Runs entirely in your browser. Requests go only to the relay you select. Inspect only resources you
